# WIKA_XD 运营诊断框架

更新时间：2026-04-04

## 1. 目标

统一 WIKA 和 XD 的经营诊断结构，但不混淆两店阶段。

输出纪律：
- 先真实数据，再做解释
- 建议必须能落到动作
- 数据缺失时明确写“暂不能判断”

## 2. 诊断结论的四种类型

以后每条正式诊断结论必须标记为：

1. 真实数据结论  
   例：官方 products 主数据总量、官方 orders list 总量、页面态已验证的市场 Top1

2. 基于结构的推断  
   例：目录过宽导致主线不聚焦、分组混乱可能稀释采购路径

3. 官方运营知识对应建议  
   例：根据官方 Product Information Score、关键词趋势、首次回复课程给出的动作建议

4. 因数据缺失暂不能判断  
   例：询盘质量、客户画像、overview 未接通时的市场结论

## 3. 诊断主线

### 3.1 店铺阶段判断
- 先判断店铺阶段，再看指标
- 不允许用同一套标准分析 WIKA 和 XD

### 3.2 产品结构诊断
- 数量是否过宽或过窄
- 分组 / 系列是否清晰
- 新品与老品是否分层
- 是否有明显长期未更新产品
- 是否存在核心包装品类缺失

### 3.3 产品管理诊断
- 标题是否清晰可检索
- 关键词是否与搜索行为匹配
- 主图 / 视频 / 详情是否支持采购决策
- 是否存在内容老化
- 是否存在 PIS / 产品质量待补项

### 3.4 订单诊断
- 订单体量与结构
- 时间分布
- 状态分布
- 国家 / 产品贡献
- 发货方式
- 是否能形成稳定分析层

### 3.5 市场与流量诊断
- UV / PV / 访客来源
- 国家来源
- 热搜关键词
- 热门产品
- 高曝光低点击 / 高点击低询盘

### 3.6 询盘与客户诊断
- 响应率 / 快速回复率
- 询盘来源
- 客户质量
- 跟进节奏
- 报价质量

## 4. WIKA 口径

WIKA 当前按**新店口径**诊断：
- 重点看基础信息完整度
- 重点看发布质量与曝光起量
- 重点看首批询盘
- 重点看响应速度
- 重点看初始产品结构是否合理

WIKA 不应过早用“成熟老店效率优化口径”压判断。

## 5. XD 口径

XD 当前按**老店 / 成熟店口径**诊断：
- 重点看历史产品沉积
- 重点看老品清理
- 重点看系列结构
- 重点看流量来源效率
- 重点看询盘来源质量
- 重点看更新策略与内容老化

XD 不应再只用“新店补基础”的口径来判断。

## 6. 输出模板

每个店铺的正式经营报告至少要包含：

1. 当前店铺阶段判断
2. 当前最突出问题
3. 当前最值得保留的优势
4. 产品结构问题
5. 流量与询盘问题
6. 店铺运营动作问题
7. 与另一店铺的差异
8. 短期行动建议（1~2 周）
9. 中期行动建议（1~2 月）
10. 长期动作建议（持续优化）

## 7. 当前执行边界

- WIKA：
  - 可以做较完整的产品 / 订单 / 市场诊断
  - 但 inquiries / customers 仍不能伪诊断
- XD：
  - 目前能做 auth/bootstrap、products 主数据、orders 最小官方层诊断
  - 不能伪造 overview / customers 结论
