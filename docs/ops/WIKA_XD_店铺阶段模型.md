# WIKA_XD 店铺阶段模型

更新时间：2026-04-04

## 1. 模型目的

统一后续所有经营诊断的阶段口径，避免：
- 用新店逻辑分析老店
- 用老店效率逻辑压新店
- 把结构问题和基础问题混在一起

## 2. 项目内阶段定义

本模型是**项目内阶段模型**，基于 Alibaba 官方卖家课程关注点，再结合 WIKA / XD 当前实际情况做适配。

### 阶段 A：新店启动期

重点：
- 产品发布完整度
- 标题、关键词、主图基础质量
- 首批曝光
- 首批询盘
- 响应速度
- 基础分组是否合理

不应过度强调：
- 大规模老品清理
- 复杂国家结构优化
- 过深的效率模型

### 阶段 B：基础起量期

重点：
- 重点产品聚焦
- 高曝光低点击识别
- 高点击低询盘识别
- RFQ 与询盘接住能力
- 内容补强

### 阶段 C：结构优化期

重点：
- 系列化管理
- 目录收口
- 老品清理
- 国家 / 市场结构优化
- 重点产品与形象产品分层

### 阶段 D：成熟效率期

重点：
- 流量来源效率
- 国家来源质量
- 询盘质量
- 订单结构
- 转化效率
- 更新节奏与内容老化治理

## 3. WIKA 当前阶段映射

WIKA 当前按**新店口径**执行。

当前诊断重点：
- 基础信息完整度
- 产品发布质量
- 曝光起量
- 首批询盘
- 响应速度
- 初始结构是否合理

WIKA 现在不应用“老店积压目录”的逻辑主导判断。

## 4. XD 当前阶段映射

XD 当前按**老店 / 成熟店口径**执行。

当前诊断重点：
- 历史产品沉积
- 目录是否过宽
- 老品清理
- 系列结构
- 流量来源效率
- 询盘来源质量
- 产品更新策略

XD 现在不应用“先补首批基础信息”的新店主逻辑主导判断。

## 5. 阶段与动作对应

### WIKA（新店）
- 短期：
  - 把标题、关键词、主图、MOQ、打样、交期等基础项拉齐
  - 优先做重点包装产品的首屏表达
- 中期：
  - 形成首批重点款与国家内容分层
  - 增加 RFQ / 询盘承接质量
- 长期：
  - 逐步从基础起量期过渡到结构优化期

### XD（老店）
- 短期：
  - 目录收口
  - 包装主线聚焦
  - 老品排查
- 中期：
  - 订单结构 / 流量来源 / 国家结构效率优化
  - 更新节奏治理
- 长期：
  - 进入成熟效率期的持续优化
