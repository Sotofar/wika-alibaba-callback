# 阿里国际站双账号运营架构

## 1. 总体架构说明

本仓库采用“共享中台 + 独立项目组”结构，并补充统一的 `Access / Auth / Callback / Token` 运行规范。

- `shared/`
  - 存放通用方法、SOP、模板、规则与共享模块
- `shared/access/`
  - 存放授权接入、callback、token 管理与排障规则
- `WIKA/`
  - 存放 WIKA 业务主线的文档、证据、脚本和项目资料
- `XD/`
  - 存放 XD 历史资料与目录归档，不作为当前业务主线

## 2. 职责边界

### 共享中台负责

- 沉淀方法论、模板、SOP 和标准字段口径
- 统一授权接入、callback、token、排障与验证方法
- 沉淀可跨账号复用的脚本和模块

### 共享中台不负责

- 存放任何单店铺专属业务数据
- 输出某个账号的结论性经营判断
- 存放真实密钥、真实 token、真实 cookie

### WIKA 项目负责

- 维护 WIKA 的真实验证记录、经营结论、阶段文档与证据
- 所有 WIKA 业务输出只落在 `WIKA/` 对应目录

### XD 项目负责

- 保留 XD 的历史资料、脚本与证据归档
- 所有 XD 业务输出只落在 `XD/` 对应目录

## 3. 目录边界规则

- 今后 WIKA 的业务工作、文档、脚本、证据、规划材料，只进入 `WIKA/` 对应目录。
- 今后 XD 的业务工作、文档、脚本、证据、规划材料，只进入 `XD/` 对应目录。
- truly shared / common 内容单独保留在根目录或 `shared/`、`src/`、`common/`，不再混放到 WIKA 或 XD 业务目录。

## 4. 命名与沉淀原则

- 方法类文档使用 `*-rules.md`、`*-framework.md`、`*-sop.md`
- 模板类文档使用 `*-template.md`
- 证据类文件进入各自 `docs/framework/evidence/`
- 运行时与共享模块优先保留在 `shared/` 或 `src/`

## 5. 扩展原则

后续如果新增第三个账号：

1. 复制项目目录结构，不改共享中台结构。
2. 复用 `shared/` 与 `src/` 中的通用能力。
3. 新账号自己的文档、脚本、证据只放进对应业务目录。
4. 只有在确认具备通用价值时，才把方法回写到共享区。
