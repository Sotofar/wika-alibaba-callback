# WIKA_XD 运营动作能力清单

更新时间：2026-04-04

## 1. 当前已具备的运营动作能力

### 两店通用
- 检查 auth/debug 状态
- 检查 bootstrap 恢复状态
- 获取官方 products 主数据
- 获取官方 orders 最小 list/detail
- 生成桌面交付报告

### WIKA 额外具备
- 基于已验证 products 表现层做重点产品诊断
- 基于已验证 orders 分析层做订单结构诊断
- 基于已验证 overview / market 做市场集中度观察

### XD 当前具备
- 基于官方 products 主数据做目录与分组诊断
- 基于官方 orders 最小 list/detail 做订单最小结构判断

## 2. 当前只可谨慎使用的能力

- WIKA 本地页面态分析结果
  - 真实，但不是生产无状态路径
  - 可用于运营判断，不可误写为 XD 的生产可复制方案
- XD orders 初步分析
  - 只可基于官方 list/detail 样本做最小结构判断
  - 不可替代完整订单分析层

## 3. 当前明确不具备的能力

- XD overview 运营诊断
- XD inquiries / messages / customers 运营诊断
- 生产可复用的自动回复询盘动作

## 4. 回复询盘入口尝试结论

- 当前没有已验证、低风险、生产可复用的官方回复询盘入口
- 已知 customer / inquiry 相关能力存在权限阻塞
- 当前不建议为了接入回复询盘而引入高风险页面态、验证码或人工交互链路
- 因此本阶段结论是：
  - 已尝试识别
  - 未形成可生产接入方案
  - 当前应停止继续推进

## 5. 下一步最有价值的动作

1. 补 `orders` 最小汇总 / 趋势层
2. 继续识别 XD overview 生产无状态数据源
3. 继续确认 XD inquiries/messages/customers 的权限与接口边界
4. 维持桌面报告与项目内状态文档同步
5. 只在获得明确官方、低风险入口时再重启“回复询盘”接入尝试
