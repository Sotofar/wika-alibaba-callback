# WIKA 产品运营任务清单

用于产品运营补齐标题、关键词、详情、图片、视频和新品方向。

## 任务清单

### P1｜补齐高优先级产品内容缺口

- 任务编号：`PROD-P1-001`
- 任务类型：`product_optimization`
- 负责人角色：产品运营
- 归属任务：任务3
- 来源报告：`WIKA/docs/reports/deliverables/WIKA_产品优化建议报告.md`
- 来源证据：`WIKA/docs/reports/deliverables/evidence/WIKA_正式运营报告包证据.json`
- 为什么重要：报告指出产品层仍有描述、关键词、素材和低分商品问题，这是当前最可控的转化基础动作。
- 预期收益：提升产品承接质量，为后续曝光、点击和询盘承接提供更稳定底盘。
- WIKA 支撑范围：`mostly_supported_needs_human_confirm`
- 是否需要人工输入：是
- 到期窗口：本周
- 不做的风险：产品内容继续拖累曝光承接和询盘转化。

执行步骤：
- 按产品优化建议报告列出优先处理产品。
- 先补描述、关键词、主图和详情结构。
- 使用 product-draft-workbench / product-draft-preview 做安全草稿准备。
- 人工确认后再进入平台内处理，不由 WIKA 直接发布。

输入要求：
- 产品规格
- 材质
- MOQ
- 定制能力
- 主图
- 详情图
- 视频素材

验收标准：
- 优先产品均完成描述和关键词补齐。
- 每个产品都有人工确认过的主图或详情素材计划。
- 未把 preview 写成平台内发布成功。

阻塞项：
- 无

### P2｜补齐产品媒体与详情页素材

- 任务编号：`PROD-P2-001`
- 任务类型：`product_media`
- 负责人角色：产品运营
- 归属任务：任务3
- 来源报告：`WIKA/docs/reports/deliverables/WIKA_产品优化建议报告.md`
- 来源证据：`WIKA/docs/reports/deliverables/evidence/WIKA_正式运营报告包证据.json`
- 为什么重要：产品内容优化不只是文案，主图、细节图、视频和详情结构会影响询盘信任。
- 预期收益：提高商品信息完整度和 B2B 买家判断效率。
- WIKA 支撑范围：`mostly_supported_needs_human_confirm`
- 是否需要人工输入：是
- 到期窗口：本周
- 不做的风险：产品优化只停留在文本层，无法支撑高质量询盘。

执行步骤：
- 按产品优先级列出缺素材产品。
- 把缺失素材交给设计或产品负责人补齐。
- 用 product-draft-preview 检查草稿完整度。

输入要求：
- 主图
- 细节图
- 应用场景图
- 视频
- 详情页结构

验收标准：
- 每个优先产品至少有主图和详情补齐计划。
- 素材经过人工确认。
- 草稿只停留在准备层，不写成发布成功。

阻塞项：
- 无

### P3｜基于现有订单贡献整理新品方向

- 任务编号：`PROD-P3-001`
- 任务类型：`new_product_direction`
- 负责人角色：产品运营
- 归属任务：任务3
- 来源报告：`WIKA/docs/reports/deliverables/WIKA_产品优化建议报告.md`
- 来源证据：`WIKA/docs/reports/deliverables/evidence/WIKA_正式运营报告包证据.json`
- 为什么重要：新品建议应基于已有产品贡献和可见需求，不应跳到无证据大类。
- 预期收益：形成更稳妥的新产品补充方向。
- WIKA 支撑范围：`mostly_supported_needs_human_confirm`
- 是否需要人工输入：是
- 到期窗口：30 天以上
- 不做的风险：新品动作可能脱离当前真实业务信号。

执行步骤：
- 从订单贡献和重点产品中整理候选新品方向。
- 人工确认供应链和利润空间。
- 进入产品安全草稿准备层。

输入要求：
- 现有订单贡献产品
- 产品利润判断
- 供应链可行性

验收标准：
- 新品方向均有来源产品或订单信号。
- 供应链可行性经过人工确认。
- 未把新品建议写成自动上架。

阻塞项：
- 无
