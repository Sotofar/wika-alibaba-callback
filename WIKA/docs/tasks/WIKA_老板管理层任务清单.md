## 任务清单

### P1｜拍板本周三件优先事项

- 任务编号：`MGMT-P1-001`
- 任务类型：`decision`
- 负责人角色：老板/管理层
- 归属任务：通用运营
- 来源报告：`WIKA/docs/reports/deliverables/WIKA_管理层简报.md`
- 来源证据：`WIKA/docs/reports/deliverables/evidence/WIKA_正式运营报告包证据.json`
- 为什么重要：stage47 报告包已经把问题收敛为产品内容、销售交接、广告与页面输入三条主线；如果管理层不拍板，执行会继续分散。
- 预期收益：把团队注意力集中到本周最能产生经营收益的动作上，减少泛泛优化和重复讨论。
- WIKA 支撑范围：`mostly_supported_needs_human_confirm`
- 是否需要人工输入：是
- 到期窗口：今天
- 不做的风险：团队会继续把报告当资料看，而不是转成执行动作。

执行步骤：
- 确认 P1 为产品内容补强、销售草稿 SOP、广告与页面输入补齐。
- 指定每条主线的负责人和本周交付时间。
- 要求运营负责人按任务总看板回填执行状态。

输入要求：
- 管理层确认本周资源优先级
- 确认产品运营、销售/跟单、店铺运营负责人

验收标准：
- 三条优先事项均有明确负责人。
- 每条事项均有本周完成窗口。
- 没有把 task3/task4/task5 写成平台内自动执行闭环。

阻塞项：
- 无

### P3｜决定是否投入广告数据长期归集

- 任务编号：`MGMT-P3-001`
- 任务类型：`decision`
- 负责人角色：老板/管理层
- 归属任务：输入层
- 来源报告：`WIKA/docs/reports/deliverables/WIKA_广告分析报告.md`
- 来源证据：`WIKA/docs/reports/deliverables/evidence/WIKA_正式运营报告包证据.json`
- 为什么重要：广告层当前最大阻塞是样本输入，不是代码；是否长期做，需要管理层明确投入。
- 预期收益：决定广告分析是否从 readiness 进入持续运营能力。
- WIKA 支撑范围：`manual_only`
- 是否需要人工输入：是
- 到期窗口：后续跟进
- 不做的风险：广告分析能力会停留在模板层。

执行步骤：
- 评估广告数据归集是否纳入固定运营流程。
- 决定由谁每周导出和校验。
- 确认是否把广告分析纳入月度复盘。

输入要求：
- 广告投入目标
- 导出责任人
- 周度复盘机制

验收标准：
- 是否长期归集有明确结论。
- 若继续，明确负责人和节奏。
- 若不继续，广告报告保持 readiness 口径。

阻塞项：
- 需要管理层资源决策
