# WIKA_人工接管规则

更新时间：2026-04-04

本文件定义 `WIKA` 写侧验证与后续写操作的最小安全边界。它不代表产品上新闭环已经打通，也不代表允许真实发布商品。

## 1. 写侧阻塞分类

| 分类代码 | 分类名称 | 说明 | 默认动作 |
| --- | --- | --- | --- |
| `permission_blocked` | 权限阻塞 | 平台明确返回权限不足，当前应用或账号无法继续执行 | 停止自动推进，记录证据并转人工 |
| `platform_no_entry` | 平台无入口 | 当前没有识别到稳定、可复用的官方入口 | 停止自动推进，保留替代方案说明 |
| `parameter_missing` | 参数缺失 | 关键入参、业务参数或必填字段不足 | 生成待补清单，等待人工补齐 |
| `media_unavailable` | 媒体能力不可用 | 素材未准备好，或媒体上传 / 转换能力不可用 | 生成素材待补清单 |
| `category_attribute_incomplete` | 类目 / 属性信息不足 | 类目树、属性结构或必填属性不完整，无法生成可靠 payload | 停止写入，只保留草稿 |
| `manual_confirmation_required` | 写操作需要人工确认 | 技术上可继续，但会影响真实线上对象 | 停止自动执行，等待人工确认 |
| `high_risk_irreversible` | 高风险或不可逆 | 无法证明存在草稿 / 测试模式，或操作难以回滚 | 禁止自动执行，必须人工决策 |

## 2. 人工接管触发条件

命中以下任一条件，必须转人工处理：

1. 真实商品发布、上架、下架或覆盖式更新。
2. 写入内容涉及价格、MOQ、交期、付款承诺、物流承诺等真实对外承诺字段。
3. 缺少类目或属性信息，导致 payload 只能靠猜测补齐。
4. 缺少可用主图、详情图、效果图或媒体上传能力。
5. 平台返回明确权限错误，需要人工判断是否申请权限或更换路径。
6. 无法证明接口存在草稿 / 测试模式，继续执行会产生不可逆副作用。

## 3. 当前允许的自动化边界

当前阶段只允许：

- 官方读接口验证
- 原始读路由上线
- 类目 / 属性 / 媒体相关支持接口的低风险验证
- 结构化产品草稿生成
- 结构化待处理 / 告警产物生成

当前明确不允许自动执行：

- 真实商品发布
- 真实线上商品修改
- 真实客户回复发送
- 真实订单创建

## 4. 结构化待处理 / 告警产物

当前项目没有强制绑定外部通知服务，因此最小要求是生成结构化待处理 / 告警产物。推荐字段：

- `account`
- `stage`
- `status`
- `severity`
- `action`
- `api_name`
- `blocker_category`
- `triggers`
- `requires_human_handoff`
- `input_summary`
- `evidence`
- `next_action`
- `created_at`

## 5. 当前阶段收口原则

1. 接口即使已经过授权层，也不等于可以直接写回平台。
2. `payload` 草稿生成，不等于产品上新完成。
3. 接口可调，不等于正式写路由可上线。
4. 只要命中高风险或不可逆条件，就必须停留在草稿 / 验证层。
