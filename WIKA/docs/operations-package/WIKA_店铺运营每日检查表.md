# WIKA 店铺运营每日检查表

## 使用方式

每天按下表执行一次。发现问题后，不要直接写平台；先记录证据、责任人和下一步动作，再由运营负责人排期。

| 检查项 |检查动作 |记录内容 |负责人 |验收标准 |
| --- |--- |--- |--- |--- |
| 首页检查 |检查首屏定位、主 banner、核心产品露出是否清楚 |发现的问题、截图位置、建议动作 |店铺运营 |每个问题都有位置和处理建议 |
| 产品露出检查 |检查主推产品是否覆盖眼镜盒、包装套装、定制能力 |缺失产品类型、排序问题 |店铺运营/产品运营 |列出需要调整的产品露出位 |
| 询盘入口检查 |检查 Contact / Inquiry / RFQ 入口是否明显 |入口位置、文案问题、跳转问题 |店铺运营 |询盘入口可被快速找到 |
| B2B 信任信息检查 |检查工厂能力、定制能力、材质、MOQ、交期是否清楚 |缺失信任信息 |产品运营 |关键信任信息有明确补充责任人 |
| 内容更新检查 |检查产品标题、关键词、详情、图片/视频是否过期 |待更新产品与字段 |产品运营 |形成产品优化工单 |
| 数据异常检查 |看 summary / diagnostic / comparison 是否有异常信号 |异常指标、来源报告、影响 |运营负责人 |异常进入任务看板 |
| 页面噪音检查 |检查是否有零售化、无关购物车、B2C 话术或干扰询盘的内容 |噪音位置、处理建议 |店铺运营 |噪音项有保留或处理结论 |
