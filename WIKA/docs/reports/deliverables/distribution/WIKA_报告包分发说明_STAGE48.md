# WIKA 报告包分发说明 STAGE48

## 1. 报告包包含什么

Stage47 已完成正式运营报告包交付，Stage48 只把它整理为可分发、可执行、可人工接手、可复跑的运营闭环。当前报告包包含 8 份 Markdown、8 份 PDF、报告包索引、评分 JSON、证据 JSON 和 PDF 清单。

8 份 PDF：

- `WIKA_管理层简报.pdf`
- `WIKA_运营周报.pdf`
- `WIKA_经营诊断报告.pdf`
- `WIKA_产品优化建议报告.pdf`
- `WIKA_广告分析报告.pdf`
- `WIKA_店铺执行清单.pdf`
- `WIKA_销售跟单使用清单.pdf`
- `WIKA_人工接手清单.pdf`

## 3. 每份报告解决什么问题

- 管理层简报：帮助管理层快速判断当前 WIKA 运营报告包能支持哪些决策、哪些结论仍需人工补数。
- 运营周报：给运营负责人提供当前已读页面范围内的订单、商品、店铺和执行信号。
- 经营诊断报告：把当前可读业务信号整理成诊断素材，但不是完整经营驾驶舱。
- 产品优化建议报告：列出需要产品运营进一步补充素材、规格、材质和关键词的方向。
- 广告分析报告：保留广告分析框架和人工补数入口，不编造广告消耗、ROI 或转化。
- 店铺执行清单：把能立刻进入人工执行的店铺运营动作拆成清单。
- 销售跟单使用清单：把可用于跟单的订单、买家和人工确认字段拆出。
- 人工接手清单：说明哪些事项必须由人工补充、确认或执行。

## 4. 哪些内容可以直接用于决策

- 是否进入固定周报 / 月报分发节奏。
- 哪些角色需要优先阅读哪份报告。
- 哪些人工输入是后续报告质量提升的关键阻塞。
- 当前是否可以把报告包交给业务侧试读。
- 当前线上 WIKA report route 是否足以支持报告包分发和 sanity 检查。

## 5. 哪些内容需要人工补数

- 真实广告导出样本。
- 页面人工盘点输入。
- 产品素材、规格、材质、关键词校对。
- 最终报价、交期、样品与买家信息。
- 订单末端人工确认字段。

## 6. 哪些内容不能误读

- 当前报告包不是 task 1 complete。
- 当前报告包不是 task 2 complete。
- 当前报告包不是 task 3 complete。
- 当前报告包不是 task 4 complete。
- 当前报告包不是 task 5 complete。
- task 6 excluded。
- no write action attempted。
- WIKA-only thread for business work。
- XD untouched in business execution。
- 这不是 full business cockpit。
- 不包含编造广告数据、GMV、转化率、国家结构或完整经营诊断。

## 7. 如何分发给不同角色

- 老板 / 管理层：优先发管理层简报和经营诊断报告。
- 运营负责人：优先发运营周报、经营诊断报告、店铺执行清单。
- 店铺运营：优先发店铺执行清单和运营周报。
- 产品运营：优先发产品优化建议报告和人工接手清单。
- 销售 / 跟单：优先发销售跟单使用清单和人工接手清单。
- 人工接手人员：优先发人工接手清单、人工补数总表和人工补数字段清单。

## 8. 如何收集反馈

每个角色反馈只收三类信息：

1. 这份报告是否能直接支持当前工作。
2. 哪些字段或结论需要人工补充后才可信。
3. 下一轮最希望报告包改进的一个点。

反馈不应要求 Codex 继续无证据试 API，也不应要求自动执行写侧动作。

## 9. 下一次复跑时怎么处理

下一次复跑应按 `WIKA_正式运营报告包_Runbook_STAGE48.md` 执行：

1. 确认 Markdown、PDF、评分 JSON、证据 JSON 和 PDF 清单存在。
2. 检查桌面副本是否存在；缺失时只记录为分发风险，不阻塞仓库报告包。
3. 执行线上 sanity。
4. 若 `operator-console` 或 `action-center` 仍 degraded，按 Stage48 的合理降级策略记录，不阻塞分发。
5. 若人工补数已完成，再复跑报告生成和 PDF 导出。
