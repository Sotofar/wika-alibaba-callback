# WIKA 反馈回收与改进规则 STAGE49

## 1. 反馈如何分类

反馈统一分为五类：

1. 报告表达问题：看不懂、太长、章节顺序不合理。
2. 业务动作问题：建议能不能执行、谁执行、何时执行。
3. 人工补数问题：缺广告样本、页面盘点、产品素材、销售反馈或订单确认。
4. 新数据源问题：需要 GMV、转化率、国家结构或完整经营诊断。
5. 不应推进问题：写侧动作、无证据 API 扩展、XD 业务执行。

## 2. 哪些反馈可以直接改报告

- 标题、摘要、章节顺序。
- 角色阅读顺序。
- 边界说明措辞。
- 人工补数说明。
- 输出格式偏好。

## 3. 哪些反馈需要人工补数

- 广告效果、ROI、投产比、关键词效果。
- 页面优化优先级。
- 产品素材、规格、材质、关键词。
- 报价、交期、样品、买家信息。
- 订单末端状态确认。

## 4. 哪些反馈需要新数据源

- 全量 GMV。
- 转化率。
- 国家结构。
- 产品贡献。
- 完整经营诊断。

这些反馈不能在当前 stage49 自动完成，只能进入后续外部证据等待队列。

## 5. 哪些反馈不应该做

- 要求 Codex 自动修改商品、订单或上下架。
- 要求发送消息。
- 要求无证据继续试 API。
- 不应要求把 degraded route 写成完全消除。
- 要求把 task1-5 写成 complete。

## 6. 下一轮如何决定优先级

优先级规则：

1. 能直接提升业务阅读和分发效率的文案/结构优化优先。
2. 能补齐广告、页面、产品、销售、订单证据的人工输入优先。
3. 需要新数据源但没有证据的需求暂缓。
4. 写侧动作和无证据 API 扩展不进入下一轮。
