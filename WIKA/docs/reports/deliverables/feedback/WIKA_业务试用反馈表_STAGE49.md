# WIKA 业务试用反馈表 STAGE49

## 使用说明

本反馈表用于收集 WIKA 正式运营报告包的业务试用反馈。请只评价报告是否能帮助当前业务工作，不要把反馈变成新的 API 探索、写侧动作或完整经营驾驶舱需求。

## 基本信息

- 填写人：
- 角色：
- 试用日期：
- 阅读过的报告：
- 是否需要固定每周收到：

## 反馈问题

1. 报告是否看得懂？请写明最难理解的章节。
2. 哪一份报告最有用？为什么？
3. 哪一份报告信息过多？哪些内容可以压缩？
4. 哪些字段需要解释？
5. 哪些建议可以执行？
6. 哪些建议不能执行？原因是什么？
7. 哪些地方需要真实广告数据？
8. 哪些地方需要页面盘点？
9. 哪些地方需要产品素材、规格、材质或关键词校对？
10. 哪些地方需要销售 / 跟单反馈？
11. 是否希望固定每周收到？
12. 是否希望改成 Excel / 飞书 / 邮件版？

## 不应反馈为“立即自动执行”的事项

- 修改商品。
- 发布 / 上下架。
- 提交草稿。
- 发送消息。
- 修改订单。
- 无证据生成 GMV、转化率、国家结构或完整经营诊断。

## 反馈后如何处理

- 可直接改报告表达的问题，进入下一轮报告文案优化。
- 需要人工补数的问题，进入 stage49 intake 模板。
- 需要新数据源的问题，等待外部证据，不进入本轮自动化。
- 写侧动作、无证据 API 扩展和 XD 任务不进入 WIKA stage49。
