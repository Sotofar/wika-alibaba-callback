# WIKA 反馈处理说明 STAGE50

## 1. 如何录入反馈

每条反馈录入 `WIKA_反馈回收台账_STAGE50.csv`，至少填写：

- `feedback_id`
- `role`
- `source_report`
- `feedback_text`
- `feedback_type`
- `priority`
- `owner`
- `status`
- `next_action`

真实反馈未收到前，保留占位文本和 `OPEN` 状态。

## 2. 如何判断优先级

- `P1`：影响报告能否被业务侧使用，或影响人工补数回收。
- `P2`：影响报告可读性、格式、角色分发效率。
- `P3`：体验优化或长期增强。

## 3. 哪些反馈可以直接改报告

- 表达不清。
- 章节顺序不合理。
- 角色阅读顺序不清。
- 边界说明不够明确。
- 输出格式需要 Excel / 飞书 / 邮件版。

## 4. 哪些反馈要等人工补数

- 广告效果和 ROI。
- 页面优化优先级。
- 产品素材、规格、材质、关键词。
- 报价、交期、样品、买家信息。
- 订单末端确认。

## 5. 哪些反馈需要新数据源

- 全量 GMV。
- 转化率。
- 国家结构。
- 产品贡献。
- 完整经营诊断。

这些反馈不能在没有新外部证据时进入自动改版。

## 6. 哪些反馈不应该做

- 要求自动修改商品、订单、草稿、上下架或发送消息。
- 要求无证据继续 API 探索。
- 要求把 task1-5 写成 complete。
- 要求把 accepted degraded route 写成完全消除。
- 要求把人工补数模板空字段写成已补齐。

## 7. 下一轮如何进入 stage51

进入 stage51 的条件：

1. 至少一个角色反馈已回收。
2. 或至少一类人工补数模板有真实输入。
3. 或业务侧明确要求调整报告格式。

如果没有反馈和补数，不应继续扩展报告体系。
