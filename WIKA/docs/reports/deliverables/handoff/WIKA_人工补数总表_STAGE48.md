# WIKA 人工补数总表 STAGE48

## 目的

Stage47 报告包已经可以分发，但部分结论仍受人工输入缺口限制。Stage48 将这些缺口拆成明确补数项，避免把“需要人工确认”的内容误写成已完成业务动作。

## 必须补的输入

| 输入区域 | 必须补什么 | 影响报告 | 补完后 Codex 能做什么 |
| --- | --- | --- | --- |
| 真实广告导出样本 | 广告消耗、展示、点击、转化、关键词或计划维度导出 | 广告分析报告 | 生成基于真实样本的广告摘要和异常点 |
| 页面人工盘点输入 | 页面标题、主图、详情、卖点、CTA、证书、FAQ 等盘点结果 | 产品优化建议报告、店铺执行清单 | 将页面优化从方向建议推进到逐页执行建议 |
| 产品素材 | 主图、细节图、证书、视频、应用场景图 | 产品优化建议报告 | 标注素材缺口和优先补图顺序 |
| 规格 | 型号、尺寸、参数、适配场景 | 产品优化建议报告、销售跟单清单 | 生成更准确的询盘和跟单问答素材 |
| 材质 | 材料、表面处理、认证、使用环境 | 产品优化建议报告 | 强化产品卖点和页面描述建议 |
| 关键词校对 | 核心词、长尾词、禁用词、目标行业词 | 产品优化建议报告、广告分析报告 | 修正关键词建议，减少无证据猜测 |
| 最终报价 | 报价区间、币种、MOQ、价格有效期 | 销售跟单使用清单 | 生成可人工发送前复核的跟单草稿素材 |
| 交期 | 标准交期、加急交期、样品交期 | 销售跟单使用清单 | 生成按买家需求分层的跟单建议 |
| 样品与买家信息 | 样品状态、买家国家、买家需求、历史沟通 | 销售跟单使用清单、人工接手清单 | 排列跟单优先级和人工确认事项 |
| 订单末端人工确认字段 | 发货状态、物流异常、付款状态、售后状态 | 经营诊断报告、人工接手清单 | 把订单末端风险从信号推进到人工处理清单 |

## 可选增强输入

- 竞品页面截图和链接。
- 重点客户清单。
- 重点产品利润区间。
- 销售团队的实际跟单反馈。
- 运营团队对页面优先级的主观排序。

## 会改变广告报告的输入

- 真实广告导出样本。
- 关键词维度效果。
- 计划 / 单元 / 创意维度消耗和转化。
- 广告侧的时间窗口说明。

没有这些输入前，广告报告只能保留为“分析框架 + 补数入口”，不能写 ROI、投产比或转化率结论。

## 会改变页面优化报告的输入

- 页面人工盘点输入。
- 产品素材、规格、材质。
- 关键词校对。
- 产品运营对目标客户和应用场景的确认。

补齐后，产品优化建议可以从保守建议推进到逐产品、逐字段的优化清单。

## 会改变销售跟单清单的输入

- 最终报价。
- 交期。
- 样品与买家信息。
- 订单末端人工确认字段。

补齐后，销售跟单使用清单可以变成更接近实际跟进动作的人工执行包。

## 会改变人工接手清单的输入

- 所有高优先级补数字段。
- 业务侧对责任人的确认。
- 业务侧对截止时间的确认。

补齐后，人工接手清单可以从“待补字段列表”推进到“责任人 + 截止时间 + 下一步动作”的执行版本。
