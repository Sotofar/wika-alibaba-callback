# WIKA 人工接手执行说明 STAGE48

## 1. 接手原则

人工接手只处理当前报告包明确标出的补数、确认和业务判断事项。不得把报告建议直接理解为已执行动作，也不得要求 Codex 自动修改商品、订单、草稿、上下架状态或发送消息。

## 2. 接手顺序

1. 先阅读 `WIKA_人工接手清单.pdf`。
2. 再阅读 `WIKA_人工补数总表_STAGE48.md`。
3. 按 `WIKA_人工补数字段清单_STAGE48.csv` 逐项补数。
4. 将补数文件放到清单中的 `where_to_store` 路径。
5. 标注责任人、补数日期、样本窗口和来源。
6. 补完后再触发下一轮报告包复跑。

## 3. 责任拆分

- 运营负责人：确认补数优先级、报告分发节奏和执行闭环。
- 店铺运营：补页面盘点、店铺执行反馈和页面问题。
- 产品运营：补产品素材、规格、材质和关键词校对。
- 广告负责人：补真实广告导出样本和广告时间窗口。
- 销售 / 跟单：补最终报价、交期、样品与买家信息。
- 人工接手人员：汇总所有待补字段并确认是否满足下一轮复跑条件。

## 4. 补完后 Codex 可以继续做什么

- 重新生成广告样本摘要。
- 重新生成页面优化建议。
- 重新生成销售跟单使用清单。
- 更新人工接手清单。
- 更新报告包分发说明和下一轮执行建议。

## 5. 何时不要继续自动推进

- 没有新的人工输入，只是想重复生成同一批 PDF。
- 需要修改商品、订单、草稿、上下架或发送消息。
- 需要判断 GMV、转化率、国家结构或完整经营诊断，但没有对应真实数据。
- 需要把 task1-5 写成 complete，但缺少真实执行证据。

## 6. 人工提交补数时的最低要求

每份补数文件至少包含：

- 数据来源。
- 时间窗口。
- 责任人。
- 生成或导出日期。
- 字段说明。
- 是否可用于报告复跑。
