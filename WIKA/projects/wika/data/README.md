# WIKA 业务数据层

## 当前阶段目标

本目录用于承接 WIKA 的业务数据读取层，和现有 access 层分开。

当前阶段只做：

- 真实数据源清单
- 店铺整体概览模块骨架
- 产品表现模块骨架
- 第一版经营分析报告模板
- 第一版建议生成逻辑

当前阶段不做：

- 修改现有 OAuth、callback、token 持久化逻辑
- 修改生产配置
- 把未读到的数据伪装成已接通

## 目录说明

- `overview/`：店铺整体概览与趋势
- `products/`：产品列表与产品表现
- `orders/`：订单数据预留
- `traffic/`：流量拆解预留
- `ads/`：广告与关键词预留
- `reports/`：报告模板
- `recommendations/`：建议生成规则

## 当前阶段真实结论

截至当前代码和验证结果：

- access 层已打通
- business data 层尚未真正读到后台业务数据
- 本目录的第一阶段交付重点是把真实数据源、模块边界、字段框架和报告框架先定下来
