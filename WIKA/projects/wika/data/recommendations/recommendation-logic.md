# WIKA 运营建议生成逻辑（第一版）

## 1. 产品运营建议

### 哪些产品该重点推

判定前提：

- 曝光高
- 点击率不低
- 询盘或订单表现优于店铺均值

建议动作：

- 增加首页、橱窗或重点入口曝光
- 保持内容更新
- 作为成交款持续承接

### 哪些产品该优化主图 / 标题 / 关键词

判定前提：

- 曝光高但点击率低

建议动作：

- 主图优先
- 标题前半段关键词优先
- 同时检查产品归类和关键词匹配

### 哪些产品该下架 / 重发 / 重做内容

判定前提：

- 长期无流量
- 无结构价值

建议动作：

- 低优先级产品清理
- 重发测试新标题或新类目
- 重新定义是否保留

## 2. 市场运营建议

### 哪些国家值得继续投放

判定前提：

- 流量、询盘、订单中至少有两项持续改善

### 哪些国家有流量但无询盘

判定前提：

- 访客明显存在
- 询盘率持续偏低

建议动作：

- 检查产品适配
- 检查市场表达方式
- 补充该市场更容易接受的内容点

## 3. 店铺运营建议

### 流量瓶颈

判定方式：

- 曝光低：优先查产品覆盖与词覆盖
- 点击率低：优先查主图和标题
- 询盘率低：优先查详情页和产品结构

## 4. 订单经营建议

当前作为框架预留，待订单模块接入后生成：

- 主要产品贡献
- 主要国家贡献
- 单一市场依赖
- 单一产品依赖
