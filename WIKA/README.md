# WIKA 目录说明

## 当前状态

- stage24 已 push 并完成最小 post-deploy 验证。
- WIKA 当前线上 summary / diagnostic / tools 路由未因目录重整而回退。
- 今后 WIKA 的业务工作、文档、脚本、证据、规划材料继续只在 `WIKA/` 对应目录内推进。

## 目录用途

- `WIKA/docs/framework/`
  - WIKA 阶段文档、基线、计划、矩阵、说明、样例
- `WIKA/docs/framework/evidence/`
  - WIKA 脱敏证据、验证摘要、route smoke 结果
- `WIKA/scripts/`
  - WIKA 阶段验证脚本、回归脚本、只读验证工具
- `WIKA/projects/wika/`
  - WIKA 项目资料、access 过程资产、业务材料与专题目录

## 主要入口

- 基线：`WIKA/docs/framework/WIKA_项目基线.md`
- 计划：`WIKA/docs/framework/WIKA_执行计划.md`
- 缺口矩阵：`WIKA/docs/framework/WIKA_面向6项任务_API缺口矩阵.md`
- 已上线能力：`WIKA/docs/framework/WIKA_已上线能力复用清单.md`

## 规则

- 今后 WIKA 的业务工作、文档、脚本、证据、规划材料，只在 `WIKA/` 对应目录内进行
- 需要共享给 XD 或通用基础设施的内容，保留在根目录、`shared/`、`src/` 或明确的 common 区域
- 所有中间进度、最终总结、验收结果、提交说明，一律使用简体中文输出
