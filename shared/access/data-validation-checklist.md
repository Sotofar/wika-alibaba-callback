# 国际站后台数据读取验证清单

## 使用方式

本清单用于在任一国际站账号接入后，按同一顺序验证“当前到底能读取什么”。

推荐每个账号都单独保存一份验证结果，不要共用一份真实记录。

## A. 接入前确认

- [ ] 当前账号 OAuth 已打通
- [ ] callback 已验证成功
- [ ] token create 已验证成功
- [ ] refresh token 已验证成功保存
- [ ] 自动续期已验证运行

## B. 运行时状态检查

- [ ] `/health` 返回 200
- [ ] `/auth/debug` 返回 200
- [ ] 如果 `/health` 与 `/auth/debug` 连续两轮都返回同类 502，当前轮次直接收口为 `BLOCKED_ENV`
- [ ] `client_id_present = true`
- [ ] `client_secret_present = true`
- [ ] `token_url` 与 `refresh_token_url` 存在
- [ ] token 存储路径可确认
- [ ] token 文件存在状态可确认
- [ ] token 已加载状态可确认
- [ ] refresh token 存在状态可确认

## C. 业务数据读取验证

### 店铺整体数据

- [ ] 是否已有读取接口
- [ ] 是否已实际调用
- [ ] 是否拿到返回值
- [ ] 是否能确认关键字段

### 产品数据

- [ ] 是否已有读取接口
- [ ] 是否已实际调用
- [ ] 是否拿到返回值
- [ ] 是否能确认产品列表或表现字段

### 询盘/客户数据

- [ ] 是否已有读取接口
- [ ] 是否已实际调用
- [ ] 是否拿到返回值
- [ ] 是否能确认询盘时间、来源产品、客户地区等字段

### 广告/投放数据

- [ ] 是否已有读取接口
- [ ] 是否已实际调用
- [ ] 是否拿到返回值
- [ ] 是否能确认消耗、点击、曝光、关键词等字段

### 其他数据

- [ ] 活动数据
- [ ] 商品诊断数据
- [ ] 页面优化相关数据

## D. 状态判定规则

- [ ] 只有“代码存在 + 实际调用成功 + 拿到返回值”时才标记为 `已验证可读取`
- [ ] 只有“代码存在但这次没实际调用”时才标记为 `代码已接好但未验证`
- [ ] 如果压根没有读取实现，标记为 `当前未接入`
- [ ] 只有拿到明确权限/开放类报错，才标记为 `权限不足或接口未开放`

## E. 证据归档

- [ ] 记录使用的接口或函数
- [ ] 记录关键文件路径
- [ ] 记录关键字段名
- [ ] 记录验证时间
- [ ] 记录限制说明
- [ ] 记录后续待接入项

## F. 给后续账号复制时的要求

- [ ] 只复制模板和流程
- [ ] 不复制真实 token
- [ ] 不复制真实 secret
- [ ] 不复制其他账号的真实验证结果
- [ ] 每个账号单独输出读取清单和验证记录
## Stage26 补充规则
- [ ] `200 + *_response` 不能直接记为通过
- [ ] 必须确认业务 payload 里存在真实字段，而不是只有 `request_id / _trace_id_`
- [ ] 空 `result` / 空 `value` / 空数组，应归类为 `NO_DATA`
- [ ] 单个对象级 `InsufficientPermission` 不能直接扩大写成“整套账号未开权”
