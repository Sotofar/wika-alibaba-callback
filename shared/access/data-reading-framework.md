# 国际站后台数据读取复用框架

## 1. 适用范围

本框架用于沉淀“国际站账号后台数据读取”这件事的通用方法，适用于：

- WIKA
- XD
- 未来新增的第 3 个国际站账号

本框架只沉淀方法、层次、验证顺序，不存放任何账号的真实敏感值。

## 2. 推荐的四层结构

### 第一层：接入可用性层

目标：

- 验证授权是否打通
- 验证 callback 是否正常
- 验证 token 是否可获得
- 验证 refresh token 是否可持续使用

典型输出：

- `/health`
- `/auth/debug`
- callback 成功记录
- token 文件/存储状态

### 第二层：运行时状态层

目标：

- 验证 token 是否已加载
- 验证 token 是否即将过期
- 验证自动续期是否生效
- 验证是否有 refresh 错误

典型输出：

- token 是否存在
- token 来源
- 下一次刷新时间
- 上一次刷新时间
- 上一次刷新错误

### 第三层：业务数据读取层

目标：

- 读取店铺整体数据
- 读取产品数据
- 读取询盘数据
- 读取广告数据
- 读取其他可用于运营分析的数据

这一层必须与前两层分开建设。

原因：

- 授权打通，不代表业务数据已打通
- token 可刷新，不代表店铺/广告/询盘接口已经接入

### 第四层：数据验证与分析层

目标：

- 把“读到了什么”做成清单
- 把“没读到什么”做成限制说明
- 把“哪些结果能支持运营动作”做成分析说明

典型输出：

- 数据来源表
- 字段映射表
- 数据验证记录
- 给运营看的简版结论

## 3. 通用判断原则

### 只有以下情况，才能标记为“已验证可读取”

- 代码中确实存在对应读取逻辑
- 已实际调用成功
- 已拿到返回值
- 已记录来源接口或函数

### 以下情况不能标记为“已验证可读取”

- 文档里提到过
- 理论上平台应该支持
- token 已经打通
- 计划以后会做

### 推荐的状态分类

- `已验证可读取`
- `代码已接好但未验证`
- `当前未接入`
- `权限不足或接口未开放`

其中最后一类只能在“实际调用过目标业务接口，并拿到明确权限类报错”后使用。

## 4. 共享层与项目层的职责边界

共享层负责：

- 读取框架
- 验证方法
- 状态分类标准
- 通用模板

项目层负责：

- 某账号的真实验证记录
- 某账号的真实读取结果
- 某账号的限制说明
- 某账号的私有配置和 token 隔离

## 5. 给 XD 的复用方式

当 XD 开始接入时，直接复用以下结构：

- 接入可用性验证
- 运行时状态验证
- 业务数据读取分类方法
- 验证结论输出模板

只替换以下内容：

- 账号名
- token 存储路径
- 配置变量实例
- 验证记录目录

不允许复制的内容：

- WIKA 的真实 token
- WIKA 的真实运行结果
- WIKA 的持久化路径真实值
